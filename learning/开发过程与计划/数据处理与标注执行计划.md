## 3. 可直接执行的命令

### Step A：Jira 数据处理（已可执行）

```bash
cd backend
source .venv/bin/activate
python ../scripts/process_jira_pipeline.py
```

输出：

- `ml/data/external/jira/processed/jira.sample.jsonl`
- `ml/data/external/jira/processed/jira.seed.auto.jsonl`

### Step B：生成待修订批次

```bash
cd backend
source .venv/bin/activate
python ../scripts/prepare_curation_batch.py --total 150
```

输出：

- `ml/data/curation/curation.batch.v1.jsonl`

### Step C：修订后做质量检查

```bash
python ../scripts/validate_seed_dataset.py --input ../ml/data/curation/curation.batch.v1.jsonl
```

输出：

- `learning/报告与演示材料/reports/curation-validation.md`

### Step D：切分训练集

```bash
python ../scripts/split_curated_dataset.py --input ../ml/data/curation/curation.batch.v1.jsonl
```

输出：

- `ml/data/final/train.jsonl`
- `ml/data/final/val.jsonl`
- `ml/data/final/test.jsonl`

---

## 4. 人工修订优先级

1. `summary`：是否表达核心结论（1-2句）
2. `action_items`：是否可执行（动词导向）
3. `risks`：是否为真实阻塞/负面影响
4. `open_questions`：是否为未决问题，避免和 action 重叠
