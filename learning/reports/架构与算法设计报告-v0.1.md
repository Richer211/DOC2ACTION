# 架构与算法设计报告 v0.1

## 1. 当前系统架构

Doc2Action 当前采用前后端分离架构：

- 前端：`Next.js` 工作台，负责输入、结果展示、引用查看、编辑导出
- 后端：`FastAPI`，负责文本预处理、结构化提取、引用映射
- 分析策略：`LLM API -> fallback rules` 双路径

主流程：

1. 用户输入文本/上传文件
2. 后端清洗文本并切分 chunk
3. 优先调用 LLM 结构化输出
4. 调用失败时降级到规则提取
5. 返回 summary/action/risk/question + source ids
6. 前端展示、人工编辑、导出

## 2. 算法策略分层

### L0 规则基线（当前可用）

- 优势：可控、稳定、零外部依赖
- 弱点：泛化能力有限，边界样本误判较多

### L1 API 模型增强（当前可切换）

- 优势：效果上限更高，快速迭代
- 弱点：受成本、延迟、key、网络影响

### L2 本地微调模型（下一阶段）

- 计划：LoRA/QLoRA 微调小中型开源模型
- 目标：减少对外部 API 依赖，形成算法能力证据

## 3. 评估体系设计

当前已开始搭建离线评估指标：

- Action F1
- Risk F1
- Question F1
- Summary Jaccard
- Citation Hit Rate
- Duplication Rate

基线评估脚本：

- `ml/eval/evaluate.py`

## 4. 下一阶段架构升级

1. 数据层升级：构建规范标注集（train/val/test）
2. 模型层升级：LoRA 训练与推理服务化
3. 检索层升级：RAG（向量检索 + 重排）
4. 系统层升级：缓存、任务队列、监控与告警

## 5. 面试可讲的核心取舍

- 先用规则与 API 跑通闭环，再逐步引入训练链路
- 通过降级策略保障服务可用性
- 通过指标与误差分析驱动优化，而非凭主观调参
