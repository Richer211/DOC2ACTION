## 4. 标准执行步骤

### Step 1：检查数据集字段

```bash
cd backend
source .venv/bin/activate
python ../scripts/load_meetingbank.py
```

### Step 2：抽样导出（先小样本）

```bash
python ../scripts/sample_meetingbank.py --split train --count 30
```

输出：

- `ml/data/external/meetingbank/processed/meetingbank.sample.jsonl`

### Step 3：转换成项目 seed 格式（弱标注）

```bash
python ../scripts/build_seed_from_meetingbank.py
```

输出：

- `ml/data/external/meetingbank/processed/meetingbank.seed.auto.jsonl`

### 可选：一键流水线

```bash
python ../scripts/process_meetingbank_pipeline.py
```

---

## 5. 为什么这是“弱标注”

MeetingBank 原始字段不完全等价于我们的目标字段（action/risk/question），所以自动转换只提供：

- summary（较可信）
- action_items（弱候选）
- risks / open_questions（先留空）

这些样本适合作为：

- 初始训练种子
- 人工标注加速输入

不建议直接作为最终高质量训练集。

---

## 6. 与 API / 本地训练的关系

你现在的路线应该是：

1. 公开数据集 + 真实脱敏数据 -> 建数据底座
2. 跑 baseline 评估（已完成）
3. 做 LoRA 微调（本地模型）
4. API 只作为对照组或 fallback，不是唯一方案

这条路线既满足工程落地，也满足算法岗对训练经验的要求。
