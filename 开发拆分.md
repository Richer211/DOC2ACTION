## 2. 开发总原则

- 先做闭环，不做扩展
- 先做能跑，再做好看
- 先做主流程，再补异常处理
- 做完一个模块就立刻测试
- **MVP 功能边界**仍以 `MVP.md` 为准；以下为 **MVP+** 例外：**Phase 7（LLM 岗位向 · 工程化）** 专为「可讲清企业级实践、支撑 LLM / 应用工程面试」而设，与产品「Must Have」解耦，按优先级逐步做即可。

---

## 3. MVP 交付目标

完成一个可演示版本，支持：

- 输入文本 / 上传 txt、md
- 自动生成 summary / action items / risks / open questions
- 查看引用原文
- 手动编辑结果
- 导出 markdown

## 3.1 已实现能力清单（MVP 主干 + LLM 方向扩展）

> 便于简历 / 面试对照：以下已在仓库中落地（细节以代码与 `learning/reports` 为准）。

### Phase 7（工程化）已落地摘要

- **请求追踪**：`X-Request-ID` + 结构化 HTTP 访问日志（JSON 一行）
- **可选 API Key**：`DOC2ACTION_API_KEY` → `X-API-Key` 或 `Authorization: Bearer`（未配置 Key 时不校验；配置后未带 Key 则 **401**）
- **限流**：`slowapi`，`DOC2ACTION_RATE_LIMIT`（默认 `60/minute`），作用于 `/analyze` 与 `/api/v1/analyze`
- **版本化路径**：`/api/v1/health`、`/api/v1/analyze`（旧路径保留）
- **业务完成日志**：`event=analyze_completed`（JSON 一行，含 `extractor_mode`、`rag_applied` 等，见 `backend/app/observability.py`）
- **异步 analyze（MVP+）**：`POST /api/v1/jobs/analyze` / `GET /api/v1/jobs/{job_id}`；无 Redis 时为内存队列，配置 **`DOC2ACTION_REDIS_URL`** + **`python -m app.rq_worker`** 为 **Redis + RQ**（多进程）
- **可选 JWT**：`POST /api/v1/auth/token` + Bearer（`docs/security-secrets.md`）
- **前端**：可选勾选「异步任务」轮询展示结果
- **CI**：`.github/workflows/ci.yml`（**ruff**、`pytest`、**tsc**、**eslint**）
- **工程文档**：`docs/security-secrets.md`（密钥与暴露面）、`docs/api-versioning.md`（版本与兼容策略）
- **前端**：若后端启用 Key，可在 **`frontend/.env.local`** 设置 **`NEXT_PUBLIC_DOC2ACTION_API_KEY`**（勿提交）；去掉则请求不带 Key，在后端启用 Key 时会 **401**（与截图一致）

### 产品与主链路

- 前后端联调：`Next.js` + `FastAPI`，`/health`、`POST /analyze`
- 抽取模式：`auto` / `rules` / `llm` / `lora`（请求头 `X-Extractor-Mode`）
- 结构化输出：`summary`、`action_items`、`risks`、`open_questions` + `source_chunk_ids`
- 前端：文档类型、输入/上传、结果编辑、引用依据、导出 Markdown

### LLM / RAG / 可观测

- **语义切块**（段落 / 句界）与环境变量、请求体 `use_semantic_chunks`
- **RAG**：OpenAI Embedding + 余弦 top-k、预算与 `rag_reason`；请求体 `use_rag` / 头 `X-RAG-Enabled`
- **LLM 实际片段**：`meta.llm_prompt_chunks` 等与 `build_prompt` 一致，前端可折叠展示；导出 Markdown 含对应章节
- **Embedding 磁盘缓存**（可选）：`DOC2ACTION_RAG_EMBED_CACHE_DIR` → SQLite，`meta.rag_embed_cache_enabled`

### 模型与评测

- **LoRA**：`ml/train/train_lora.py`、adapter 推理、后端 `DOC2ACTION_LORA_*` 环境变量
- **离线评测**：`ml/eval/evaluate.py`，含 `--rag-enabled`、`--semantic-chunks`、**`--compare-llm-rag`**（无检索 vs RAG 双跑对比报告）

### 文档与叙事

- `learning/reports/lora-phase-summary.md`：LoRA 阶段小结（面试叙事）
- `backend/.env.example`：环境变量示例（含可选 RAG 缓存说明）

## Phase 7：LLM 岗位向 · 企业级工程化补强（MVP+）

> **目标岗位**：LLM 应用 / AI 工程 / 偏后端的 Applied AI。  
> **目标**：不一定堆业务功能，但补齐**面试高频、真实服务常见**的非功能能力；每项做到「能演示、能画图、能讲权衡」即可。  
> **说明**：与 `MVP.md` 第一版边界不冲突——此处为 **求职与工程可信度** 的增量。

### 7.1 鉴权（Authentication / Authorization）

- [x] 保护 `POST /analyze`、`POST /api/v1/analyze`：**可选**环境变量 `DOC2ACTION_API_KEY`；支持 **`X-API-Key`** 与 **`Authorization: Bearer`**（未配置 Key 时本地开发行为与之前一致）
- [x] **JWT（Demo）**：配置 **`DOC2ACTION_JWT_SECRET`** 后 **`POST /api/v1/auth/token`**（演示账号 env 可改）；受保护路由接受 **Bearer JWT**（与 API Key 可并存，见 `deps_auth`）
- [x] 文档中单独成文：密钥轮换、为何不裸奔公网 → **`docs/security-secrets.md`**（可结合 `backend/.env.example` 与面试口述）
- **面试话术**：API 暴露面、最小权限、与 LLM Key 分离

### 7.2 请求追踪（Request tracing）

- [x] 每个请求生成 **`request_id`**（无则 UUID），响应头 **`X-Request-ID`**；可接受客户端传入的 `X-Request-ID`
- [x] **401** 等错误的 `detail` 中带 **`request_id`**（`deps_auth`）
- [x] 访问日志一行 JSON 含 **`request_id`**
- **面试话术**：分布式追踪入门、排障路径

### 7.3 结构化日志（Structured logging）

- [x] HTTP 层统一一行 JSON：`event, request_id, method, path, status_code, latency_ms`（`doc2action.http`）
- [x] 将 **`extractor_mode` / `rag_applied`** 等写入日志：`event=analyze_completed`（路由内 `log_analyze_completed`，字段来自响应 `meta`）
- [x] 本地开发可读（stdout JSON 行），便于后续接 ELK / Grafana（可不真接）
- **面试话术**：可观测性、如何从日志定位 LLM 超时与 RAG 退化

### 7.4 基础限流（Rate limiting）

- [x] 按 **客户端 IP** 对 **`/analyze` 与 `/api/v1/analyze`** 限流（**slowapi**，默认 `DOC2ACTION_RATE_LIMIT=60/minute`）
- [x] 超限 **429**（slowapi 默认 `Retry-After`）
- **面试话术**：成本保护、防滥用、与 OpenAI 配额的关系

### 7.5 异步长任务（可选加分）

- [x] 长文档：**`POST /api/v1/jobs/analyze`** 入队并返回 **`job_id`**，**`GET /api/v1/jobs/{job_id}`** 查询状态与结果
- [x] **未设置 `DOC2ACTION_REDIS_URL`**：进程内内存 + `BackgroundTasks`；**设置后**：**Redis + RQ**（`python -m app.rq_worker`，队列名 `DOC2ACTION_RQ_QUEUE` 默认 `doc2action`），可多进程消费
- [x] 避免同步 HTTP 长时间阻塞（同步 `/analyze` 仍保留）
- **面试话术**：超时、解耦、削峰

### 7.6 CI（Continuous Integration）

- [x] 流水线：**后端 `pytest`**（含 `tests/test_jobs.py` 等）、**前端 `tsc --noEmit`**（`.github/workflows/ci.yml`）
- [x] **ruff**（`ruff check app tests`）与 **eslint**（`npm run lint`）
- [x] PR / push 自动跑（GitHub Actions）
- **面试话术**：如何防回归、LLM 相关用 mock / 契约测

### 7.7 环境与密钥（Environment & secrets）

- [x] `backend/.env.example` 补充 **`DOC2ACTION_API_KEY`**、**`DOC2ACTION_RATE_LIMIT`**；`frontend/.env.local.example` 补充可选 **`NEXT_PUBLIC_DOC2ACTION_API_KEY`**
- [x] `.gitignore` 已忽略 **`backend/.cache/`**（embedding 缓存目录）
- [x] 密钥由部署平台注入、非写死在镜像（说明见 **`docs/security-secrets.md`**「部署时密钥注入」与容器/PaaS 小节）
- **面试话术**：合规意识、曾暴露 Key 的复盘与轮换

### 7.8 OpenAPI 与版本意识

- [x] 提供 **`GET /api/v1/health`**、**`POST /api/v1/analyze`**（与根路径 **`/health`、`/analyze`** 并存，便于渐进迁移）
- [x] OpenAPI **tags**（health / analyze / jobs）
- [x] 文档中写明 **breaking change 策略** → **`docs/api-versioning.md`**（字段新增兼容、废弃周期；亦可摘要进 PRD）
- **面试话术**：前后端 / 第三方对接 LLM 网关时的契约管理

### Phase 7 完成标准（整体）

- 能在 **5 分钟**内 demo：**带 Key 或本地关闭 Key** → **同步 analyze** → 日志里能搜到 **`request_id`** 与 **`analyze_completed`** → （可选）**`POST /api/v1/jobs/analyze`** + 轮询 **`GET /api/v1/jobs/{id}`** → 故意触发 **429** 或展示限流配置 → 指向 **CI 绿勾**
- 简历上可写一条：**「在 LLM 抽取服务中落地鉴权、可观测、限流与 CI」**（按实际完成的子集写）

## 8. 当前版本结论

MVP 主链路应以「输入 → 分析 → 展示 → 引用 → 导出」为主轴；**Phase 7** 为 **LLM 岗位向** 的工程化增量，与主轴并行推进即可。

一句话总结：

**产品闭环要稳，工程化要能讲——面试时既展示 RAG/LoRA/评测，也展示鉴权、可观测与 CI。**
